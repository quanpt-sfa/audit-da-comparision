# Final Results contract v4

This workflow supersedes the legacy `31`--`33` Results runners for the submission
bundle. Core Chapter 4 results and supplemental diagnostics are separate
reproducible workflows.

## Core timing contract

The available source panel spans 2015--2025 and uses one-year lagged accounting
inputs.

- 2015 is source support only;
- 2016 is the first model-complete audited training year;
- 2017 is the first model-based test year;
- direct preliminary/audited comparisons retain 2016--2025.

The pre-run audit verifies complete 2016 training rows separately for every
Jones-family model.

## Core corrections

1. Test cases come from the locked nonfinancial panel; estimation history comes
   from the unrestricted nonfinancial panel.
2. Jones-family models use no ordinary intercept and no feature mean-centering.
3. Historical fitting data may be winsorised; current-state `ta_scaled` is never
   clipped.
4. Fixed-reference attribution has exactly two players: PAT and CFO.
5. Direct switching uses one common complete-case population.
6. Signed shifts are reassigned within fiscal year.
7. Paired-difference regressions are primary; stacked models interact every
   state-specific slope.
8. Signed-DA state differences create one unique test per focal variable.

## Supplemental workflow

The prior placeholder paths had no committed producer and have been removed.
Supplemental results are generated by:

```powershell
pytest -q tests\test_supplemental_diagnostics.py

python .\scripts\36_run_supplemental_diagnostics.py `
  --config .\config\supplemental_diagnostics.yaml
```

The runner derives line-item revision concentration and matched near-zero CFO
randomisation from declared source data and writes a deterministic manifest to:

```text
artifacts/supplemental_diagnostics/
```

See `docs/supplemental_diagnostics.md` for the estimands and null designs.
